# init-deep 계획 및 실행결과

## 목적

`omo:init-deep` 방식으로 My Bookstation의 계층형 규칙 문서를 보강한다. 목표는 README나 개발문서의 반복이 아니라, 다음 작업자가 정본 위치와 검증 surface를 빠르게 찾도록 탐색 비용을 줄이는 것이다.

## 적용 원칙

- `--create-new` 방식은 사용하지 않는다.
- 기존 root `AGENTS.md`와 `CLAUDE.md`는 보존하고 필요한 섹션만 추가한다.
- 프로젝트 규칙에 따라 `AGENTS.md`를 바꾸면 같은 의미로 `CLAUDE.md`도 함께 바꾼다.
- `uploads_for_ai_studio/`, `temp/`, `docs/temp/`, `library-checker/temp/`, `library-checker/.wrangler/`, `dist/`, `node_modules/`는 규칙 생성 판단에서 제외한다.
- 하위 규칙은 실제 작업 경계가 분명한 디렉터리에만 추가한다.

## 실행 계획

| 단계 | 계획 | 완료 기준 | 검증 방법 |
|---|---|---|---|
| 1 | 현재 구조와 기존 root 규칙 확인 | 기존 root 규칙과 주요 코드 집중 영역이 확인된다 | `git status`, 기존 `AGENTS.md`/`CLAUDE.md`, 파일 수·라인 수 측정 |
| 2 | 비교용 root 초안 생성 | `temp/20260709_01_init-deep-root-AGENTS.generated.md`가 생성된다 | 파일 존재와 heading 확인 |
| 3 | root 규칙 선별 반영 | root 문서에 정본 위치, 저장 데이터, 검증, 탐색 제외 규칙만 추가된다 | root diff 확인 |
| 4 | 하위 규칙 추가 | `components`, `stores`, `library-checker`에 `AGENTS.md`/`CLAUDE.md` 쌍이 추가된다 | 파일 존재와 쌍별 동일성 확인 |
| 5 | 검수와 보완 | 크리티컬 이슈가 없고 경로·동기화·시크릿 검사가 통과한다 | `git diff --check`, 참조 경로 검사, 시크릿 패턴 검사 |

## 실행 결과

### Root 보강

`AGENTS.md`와 `CLAUDE.md`에 다음 4개 섹션을 같은 의미로 추가했다.

- `작업별 정본 위치`
- `저장 데이터 규칙`
- `검증 기준`
- `탐색 제외 경로`

반영하지 않은 항목:

- 긴 구조도
- 명령어 목록 반복
- README·`docs/DEVELOPMENT.md`와 중복되는 일반 설명
- code map 표 전체

### 하위 규칙 추가

| 위치 | 추가 파일 | 역할 |
|---|---|---|
| `components/` | `AGENTS.md`, `CLAUDE.md` | UI, 모달, 내 서재 화면, 검색 화면의 작업 위치와 검증 기준 |
| `stores/` | `AGENTS.md`, `CLAUDE.md` | Supabase 저장, 내 서재 상태, 설정, 자동 태그 저장 계약 |
| `library-checker/` | `AGENTS.md`, `CLAUDE.md` | Cloudflare Worker 조회 런타임, 배포·검증 surface 분리 |

### 보완 사항

검수 중 root 규칙의 `.wrangler/` 표기가 실제 위치를 충분히 드러내지 않는 문제가 있었다. 루트 경로로 오해하지 않도록 `library-checker/.wrangler/`로 수정했다.
