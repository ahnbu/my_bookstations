# 검색결과 원형 빠른추가 취소 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 도서 검색 결과 팝업의 표지 우상단 빠른 추가 UI를 E안의 선 없는 원형 버튼으로 바꾸고, 이번 팝업에서 방금 추가한 도서만 체크 클릭으로 추가 취소할 수 있게 한다.

**Architecture:** `BookSearchListModal` 안에서 이번 팝업 세션 중 빠르게 추가한 도서의 `isbn13 -> bookId`를 로컬 `Map`으로 추적한다. 미추가 도서는 `PlusIcon`, 방금 추가한 도서는 `CheckIcon` 취소 가능 상태, 원래부터 서재에 있던 도서는 `CheckIcon` 잠금 상태로 분리한다. 기존 `addToLibrary(book)`와 `removeFromLibrary(id)`를 재사용하며 Supabase 저장 구조는 바꾸지 않는다.

**Tech Stack:** React, TypeScript, Zustand, Supabase, Vite, Node test, Playwright

---

## 범위

포함한다.

- 검색 결과 표지 우상단 버튼을 E안의 선 없는 원형 버튼으로 변경한다.
- 원형 버튼은 `44px x 44px` 클릭 영역을 유지한다.
- 원형 버튼에는 border를 두지 않는다.
- 미추가 도서는 흰색 `PlusIcon`으로 표시한다.
- 이미 추가된 도서는 초록색 `CheckIcon`으로 표시한다.
- 이번 팝업에서 방금 추가한 도서는 체크 버튼 클릭으로 추가 취소할 수 있다.
- 원래부터 서재에 있던 도서는 체크로 보이지만 클릭 삭제는 불가하다.
- 카드 본문 클릭은 기존처럼 상세 보기 흐름을 유지한다.
- `event.stopPropagation()`으로 원형 버튼 클릭과 카드 클릭을 분리한다.
- 기존 소스 테스트와 브라우저 UI 테스트를 새 동작에 맞게 갱신한다.

포함하지 않는다.

- 체크박스 기반 다중 선택
- 검색 결과 카드 전체 레이아웃 개편
- Supabase `user_library` 테이블 구조 변경
- 기존 서재 데이터 재처리
- 원래부터 있던 서재 도서를 검색 팝업에서 삭제하는 기능

## 저장 데이터 영향

영향받는 저장 대상은 기존 Supabase `user_library` 테이블이다.

- 새 도서 추가는 기존 `addToLibrary(book)` 경로를 사용한다.
- 방금 추가한 도서의 취소는 기존 `removeFromLibrary(id)` 경로를 사용한다.
- 저장 payload, 테이블 컬럼, 기존 데이터 의미는 변경하지 않는다.
- 기존 데이터 재처리는 필요 없다.
- 새로 추가되는 `quickAddedBookMap`은 브라우저 메모리의 로컬 UI 상태이며 영구 저장하지 않는다.

검증 기준:

- `+` 클릭 후 `user_library` insert 경로가 기존과 동일하게 실행된다.
- 방금 추가한 도서의 체크 클릭 후 기존 `removeFromLibrary(id)`가 호출된다.
- `removeFromLibrary(id)` 호출 뒤 실제 ISBN이 서재에서 사라진 경우에만 `quickAddedBookMap`에서 해당 ISBN을 제거한다.
- 원래부터 있던 도서의 체크 클릭은 삭제를 실행하지 않는다.
- 모달을 닫으면 이번 팝업 세션의 `quickAddedBookMap`은 초기화된다.

    fetchUserLibrary: async () => undefined,

        return {
          myLibraryBooks: state.myLibraryBooks.filter(book => book.id !== id),
          myLibraryIsbnSet: nextIsbnSet,
          removeCallCount,
        };
      });

  });

  useAuthStore.setState({
    session: {
      user: {
        id: 'quick-add-test-user',
        email: 'quick-add@example.com',
      },
    },

      id: 'quick-add-test-user',
      email: 'quick-add@example.com',

- [ ] **Step 4: `handleQuickAddClick`을 추가/취소 분기형으로 바꾼다**

기존 `handleQuickAddClick` 전체를 다음으로 교체한다.

```ts
const handleQuickAddClick = async (
  event: React.MouseEvent<HTMLButtonElement>,
  book: AladdinBookItem,
) => {
  event.stopPropagation();

  const normalizedIsbn = (book.isbn13 || '').toString().trim();
  if (!session || !normalizedIsbn || addingIsbnSet.has(normalizedIsbn) || removingIsbnSet.has(normalizedIsbn)) return;

  const quickAddedBookId = quickAddedBookMap.get(normalizedIsbn);

  if (quickAddedBookId) {
    setRemovingIsbnSet(prev => new Set(prev).add(normalizedIsbn));
    try {
      await removeFromLibrary(quickAddedBookId);
      const isStillInLibrary = useBookStore.getState().isBookInLibrary(normalizedIsbn);
      if (!isStillInLibrary) {
        setQuickAddedBookMap(prev => {
          const next = new Map(prev);
          next.delete(normalizedIsbn);
          return next;
        });
      }
    } finally {
      setRemovingIsbnSet(prev => {
        const next = new Set(prev);
        next.delete(normalizedIsbn);
        return next;
      });
    }
    return;
  }

  if (isBookInLibrary(normalizedIsbn)) return;

  setAddingIsbnSet(prev => new Set(prev).add(normalizedIsbn));
  try {
    const addedBook = await addToLibrary({ ...book, isbn13: normalizedIsbn });
    if (addedBook) {
      setQuickAddedBookMap(prev => new Map(prev).set(normalizedIsbn, addedBook.id));
    }
  } finally {
    setAddingIsbnSet(prev => {
      const next = new Set(prev);
      next.delete(normalizedIsbn);
      return next;
    });
  }
};
```

## 구현 시 주의사항

- 실제 앱 코드는 이 계획 승인 후에만 수정한다.
- 신규 패키지는 설치하지 않는다.
- 기존 `addToLibrary(book)`와 `removeFromLibrary(id)`를 재사용한다.
- `removeFromLibrary(id)`는 현재 실패 시 내부에서 UI 상태를 롤백하고 throw하지 않는다. 따라서 modal 쪽은 호출 뒤 `useBookStore.getState().isBookInLibrary(normalizedIsbn)`로 실제 삭제 여부를 확인한 다음 `quickAddedBookMap`을 갱신한다.
- 원래부터 서재에 있던 책은 검색 팝업에서 삭제할 수 없게 한다.
- 방금 추가한 책만 같은 팝업 세션 안에서 취소할 수 있게 한다.
- 모달이 닫히면 `quickAddedBookMap`을 초기화해 다음 검색 세션에서 의도치 않게 삭제 가능 상태가 남지 않게 한다.
- 카드 본문 클릭과 원형 버튼 클릭은 반드시 분리한다.
- 커밋은 사용자가 요청할 때만 `cp` 스킬로 진행한다.
