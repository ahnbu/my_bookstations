# 도서검색결과 표지 위 추가 버튼 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 도서 검색 결과 팝업에서 카드 상세 보기 흐름은 유지하면서, 표지 오른쪽 위의 투명 클릭 영역 아이콘으로 같은 검색 결과에서 여러 권을 연속 추가할 수 있게 한다.

**Architecture:** `addToLibrary`가 전역 `selectedBook`에만 의존하지 않도록 선택적 `book` 인자를 받게 하되, 기존 상세 화면 호출은 그대로 동작하게 유지한다. `BookSearchListModal`은 검색 결과 카드 본문 클릭과 표지 위 추가 버튼 클릭을 분리하고, `myLibraryIsbnSet` 기반 중복 상태로 `PlusIcon`/`CheckIcon`을 표시한다.

**Tech Stack:** React, TypeScript, Zustand, Supabase, lucide-react, Vite

---

## 범위

포함한다.

- 검색 결과 팝업의 각 도서 표지 오른쪽 위에 `PlusIcon` 또는 `CheckIcon`을 표시한다.
- 아이콘 버튼은 배경과 테두리 없이 보이지만, 실제 클릭 영역은 44px x 44px로 확보한다.
- 아이콘 버튼 클릭 시 팝업은 닫히지 않고 해당 도서만 서재에 추가된다.
- 로그인하지 않은 사용자는 즉시 추가 버튼을 누를 수 없고, 버튼의 접근성 라벨과 title은 `로그인 후 추가` 상태를 드러낸다.
- 카드 본문 클릭은 기존처럼 도서 상세 보기 흐름을 유지한다.
- 이미 추가된 도서는 체크 아이콘으로 표시하고 클릭되지 않는다.
- `addToLibrary(book?)` 형태로 직접 추가 진입점을 만든다.
- 기존 `BookDetails`의 `addToLibrary()` 호출은 계속 동작한다.
- 대량등록의 임시 `selectedBook` 변경/복원 우회도 같은 직접 추가 API로 정리한다.

포함하지 않는다.

- 검색 결과 카드 전체 디자인 개편
- 체크박스 기반 다중 선택/일괄 추가
- 실제 알라딘 검색 결과 데이터 변경
- Supabase 테이블 구조 변경
- 기존 서재 데이터 재처리

## 저장 데이터 영향

영향받는 저장 대상은 Supabase `user_library` 테이블이다.

- 기존 insert payload의 의미는 유지한다.
- `book_data`, `title`, `author`, `user_id` 저장 규칙은 변경하지 않는다.
- 기존 데이터 재처리는 필요 없다.
- 새 로직은 동일한 도서를 `selectedBook` 경유 없이 직접 insert할 수 있게 하는 진입점 변경이다.
- 중복 방지는 기존 `myLibraryIsbnSet`/`isBookInLibrary` 기준을 유지한다.

검증 기준:

- 새 도서 1권 추가 시 `myLibraryBooks` 맨 앞에 추가된다.
- `myLibraryIsbnSet`에 해당 ISBN13이 추가된다.
- 같은 ISBN13의 버튼은 추가 후 체크 상태로 바뀐다.
- 중복 도서 클릭 시 새 insert가 발생하지 않는다.

## 수정 파일 구조

- Modify: `stores/useBookStore.ts`
  - `BookState.addToLibrary` 시그니처를 `addToLibrary(bookToAdd?: AladdinBookItem | SelectedBook)`로 확장한다.
  - 내부 대상 도서를 `bookToAdd ?? selectedBook`으로 결정한다.
  - 직접 전달된 도서를 추가할 때는 `selectedBook`을 지우지 않는다.
  - 기존 호출 호환성을 유지한다.

- Modify: `components/BookSearchListModal.tsx`
  - `PlusIcon`, `CheckIcon`을 사용한다.
  - 표지 이미지를 `relative` wrapper로 감싼다.
  - 표지 오른쪽 위에 44px 투명 버튼을 둔다.
  - 버튼 클릭은 `event.stopPropagation()`으로 카드 상세 보기와 분리한다.
  - 추가 중인 ISBN은 로컬 `Set`으로 관리해 중복 클릭을 막는다.

- Modify: `components/BulkBookSearchContent.tsx`
  - 새 `addToLibrary(book)` API를 사용해 임시 `selectedBook` 변경/복원 우회를 제거한다.
  - 이 변경은 검색 결과 팝업 기능과 같은 관심사인 “직접 도서 추가 API 정리”에 한정한다.

- [ ] **Step 2: 추가 대상 도서를 `bookToAdd ?? selectedBook`으로 결정한다**

`addToLibrary` 시작부를 다음 구조로 바꾼다.

```ts
addToLibrary: async (bookToAdd) => {
  const { selectedBook, isBookInLibrary } = get();
  const { session } = useAuthStore.getState();
  const targetBook = bookToAdd ?? selectedBook;

  if (!targetBook || !session || !('isbn13' in targetBook)) return null;

  const normalizedIsbn = (targetBook.isbn13 || '').toString().trim();
  if (!normalizedIsbn) {
    useUIStore.getState().setNotification({ message: 'ISBN 정보가 없어 서재에 추가할 수 없습니다.', type: 'error' });
    return null;
  }

  if (isBookInLibrary(normalizedIsbn)) {
    useUIStore.getState().setNotification({ message: '이미 서재에 추가된 책입니다.', type: 'warning' });
    return null;
  }
```

- [ ] **Step 3: 빠른 추가 핸들러를 만든다**

`handleBookClick` 아래에 다음 핸들러를 추가한다.

```ts
const handleQuickAddClick = async (
  event: React.MouseEvent<HTMLButtonElement>,
  book: AladdinBookItem,
) => {
  event.stopPropagation();

  const normalizedIsbn = (book.isbn13 || '').toString().trim();
  if (!session || !normalizedIsbn || isBookInLibrary(normalizedIsbn) || addingIsbnSet.has(normalizedIsbn)) return;

  setAddingIsbnSet(prev => new Set(prev).add(normalizedIsbn));
  try {
    await addToLibrary({ ...book, isbn13: normalizedIsbn });
  } finally {
    setAddingIsbnSet(prev => {
      const next = new Set(prev);
      next.delete(normalizedIsbn);
      return next;
    });
  }
};
```

- [ ] **Step 4: 카드 title을 상세 보기 기준으로 바꾼다**

기존 카드의 `title` 값:

```tsx
title={isDuplicate ? '이미 서재에 추가된 책입니다. 클릭하면 상세 정보를 볼 수 있습니다.' : '클릭하여 서재에 추가'}
```

다음으로 바꾼다.

```tsx
title={isDuplicate ? '이미 서재에 추가된 책입니다. 클릭하면 상세 정보를 볼 수 있습니다.' : '클릭하여 상세 정보 보기'}
```

- [ ] **Step 5: 표지 이미지를 wrapper로 감싸고 투명 버튼을 올린다**

기존 이미지:

```tsx
<img src={book.cover.replace('coversum', 'cover')} alt={book.title} className="w-32 h-48 object-cover rounded shadow-md mb-4" />
```

다음 구조로 바꾼다.

```tsx
<div className="relative w-32 h-48 mb-4">
  <img
    src={book.cover.replace('coversum', 'cover')}
    alt={book.title}
    className="w-32 h-48 object-cover rounded shadow-md"
  />
  <button
    type="button"
    onClick={(event) => handleQuickAddClick(event, book)}
    disabled={!session || isDuplicate || addingIsbnSet.has(normalizedIsbn)}
    aria-label={!session ? `${book.title} 로그인 후 추가` : isDuplicate ? `${book.title} 추가 완료` : `${book.title} 내 서재 추가`}
    title={!session ? '로그인 후 추가' : isDuplicate ? '추가 완료' : '내 서재 추가'}
    className={`absolute top-0.5 right-0.5 w-11 h-11 flex items-center justify-center transition-opacity hover:opacity-80 focus:outline-none focus:ring-2 focus:ring-blue-500 focus:ring-offset-2 focus:ring-offset-gray-900 ${
      isDuplicate
        ? 'text-green-400 cursor-default'
        : !session
          ? 'text-white/40 cursor-not-allowed'
          : 'text-white'
    }`}
  >
    {isDuplicate ? (
      <CheckIcon className="w-5 h-5 drop-shadow-md" />
    ) : (
      <PlusIcon className="w-5 h-5 drop-shadow-md" />
    )}
  </button>
</div>
```

주의: 버튼에는 배경색, border, rounded 배경 장식을 추가하지 않는다. 보이는 요소는 아이콘뿐이어야 한다.
