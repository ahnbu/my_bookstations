## 1. 배경과 목표

### 왜 Lite인가 (Full 제외 근거)
| 기대했던 효과 | 실제 | 결론 |
|---|---|---|
| 구글 로그인 빨라짐 | SW는 정적 자산만 캐싱, 인증 왕복은 대상 아님 | Full로도 해결 안 됨 |
| 오프라인에서 DB 사용 | 기본 SW는 DB 미포함(껍데기만). 오프라인 쓰기·충돌관리는 별도 프로젝트급 | 범위 외 |
| 캐시 무효화 관리 부담 | 배포마다 낡은 캐시 잔존 위험 상시 | Lite는 이 부담 없음 |
| 홈 화면 아이콘·전체화면 앱 느낌 | manifest+meta+아이콘으로 달성 | **Lite로 충분** |

→ 이 서비스(온라인 도서 재고 검색)는 오프라인·푸시 실익이 없어 **Lite가 목적 적합**.

### 목표 (완료 시 얻는 것)
- 홈 화면 추가 시 **전용 아이콘(A안: 인디고 배경 + 흰 book-open 심볼)** 표시
- 실행 시 **주소창 없는 전체화면(standalone)** 앱 느낌
- iOS/Android 홈 화면 추가 지원 (splash·테마색 반영)

## 2. 확정 사항

- **아이콘 디자인**: A안 확정 — 배경 `#5b57e0`(앱 primary 인디고), 심볼 흰색 book-open(기존 favicon 모양 재사용), 여백 포함.
- **아이콘 생성 방법**: **sharp 임시 설치** 확정 — devDependency로 `sharp` 설치 → SVG→PNG 변환 스크립트(`.mjs`) 실행 → PNG 산출 후 sharp 제거(프로젝트 의존성에 남기지 않음). 설치 전 `security-gate install-vet` 통과.
- **파일 배치**: **`public/` 폴더 배치** 확정 — Vite는 `public/` 내 파일을 빌드 시 루트로 자동 복사·서빙을 보장. 루트 직접 배치는 dist 미포함 리스크가 있어 배제. 아이콘 3종 + `manifest.webmanifest`를 `public/`에 둔다.
  - 단, `favicon.svg`는 기존 루트 참조(`/favicon.svg`)가 동작 중이므로 이동하지 않고 현행 유지(범위 밖 파일 불변경 원칙).
- **참조 경로**: `index.html` `<head>`에 절대경로(`/manifest.webmanifest`, `/icon-192.png` 등)로 추가. `public/` 파일은 빌드 후 루트에서 서빙되므로 경로는 절대경로 그대로 유효.

### 작업 1: 앱 아이콘 PNG 생성 (A안)
- **산출물**: `public/`에 아래 3개 파일
  - `public/icon-192.png` (192×192, 일반 아이콘)
  - `public/icon-512.png` (512×512, 스플래시·고해상)

- **소스**: A안 SVG(배경 라운드 사각 + 흰 book-open 심볼)를 정의 → PNG 래스터화
- **생성 방법 (확정)**: `sharp` 임시 설치.
  1. `node C:\Users\ahnbu\.claude\skills\_shared\security-gate.mjs install-vet --target sharp` 통과 확인
  2. `pnpm add -D sharp`로 임시 설치
  3. 변환 스크립트(scratchpad에 `gen-pwa-icons.mjs`)로 A안 SVG → 3종 PNG를 `public/`에 생성
  4. 산출물 검증 후 `pnpm remove sharp`로 제거 (프로젝트 의존성에 남기지 않음)
- **A안 SVG 정의 (스크립트에 고정)**: 512×512 캔버스 기준.
  - 일반(192/512): 배경 라운드 사각 `<rect>` 채움 `#5b57e0`, radius ≈ 캔버스의 22%; 중앙에 흰색(`#ffffff`) stroke book-open 심볼(lucide `book-open` path 2개), stroke-width 스케일 비례, 심볼 폭 ≈ 캔버스의 62%.
  - maskable(512): 동일하되 심볼을 중앙 안전영역(캔버스 66% 이내)으로 축소 배치. 배경은 캔버스 전체를 `#5b57e0`로 꽉 채움(모서리 라운드 없이 — OS가 마스킹).
  - book-open path: `M2 3h6a4 4 0 0 1 4 4v14a3 3 0 0 0-3-3H2z` / `M22 3h-6a4 4 0 0 0-4 4v14a3 3 0 0 1 3-3h7z` (24×24 viewBox 기준, 스케일 적용).
- **완료 기준**:
  - `public/`에 3개 PNG 파일이 존재하고, 각 파일의 실제 픽셀 크기가 192/512/512로 확인된다.
  - `icon-maskable-512.png`는 가장자리가 잘려도 심볼이 온전히 보이는 여백을 가진다(안전영역 내 심볼 배치, 심볼 ~66%).
  - `package.json`에 sharp가 남아있지 않다.
- **검증** (객관·자동 검증만):
  - `file public/icon-192.png public/icon-512.png public/icon-maskable-512.png` 출력에 각 PNG의 해상도가 192×192 / 512×512 / 512×512로 나타난다.
  - Node로 각 PNG의 실제 픽셀 크기를 확인한다(sharp 제거 전 `sharp(file).metadata()`로 width/height 출력, 또는 제거 후 PNG 헤더 파싱). 출력이 기대 해상도와 일치한다.
  - 배경색 검증: 생성 스크립트가 좌상단(배경) 픽셀 RGB를 출력하고, `#5b57e0`(≈ rgb 91,87,224) 근사값과 일치한다. (디자인 육안 확인은 **검증 불가** — 자동 판정 대상에서 제외.)

## 5. 리스크 / 주의
- **Vercel 정적 서빙**: `public/` 배치로 Vite가 빌드 시 루트로 자동 복사·서빙을 보장하므로 dist 미포함 리스크는 제거됨. 로컬 `vite build` 후 `dist/`에 아이콘·manifest가 복사됐는지 확인해 이중 검증한다.
- **sharp 설치**: devDependency 임시 추가는 `pnpm` 사용, 설치 전 `security-gate install-vet` 통과 필요. 산출 후 `pnpm remove sharp`로 제거해 의존성에 남기지 않는다.
- **아이콘 캐시**: 홈 화면에 이미 추가한 사용자는 OS 아이콘 캐시로 즉시 안 바뀔 수 있음(재추가 필요). 신규 문제 아님.
- **favicon.svg 유지**: 기존 루트 `favicon.svg`는 이동·삭제하지 않고 현행 유지(브라우저 탭 아이콘). PWA 아이콘(PNG)과 병존.
