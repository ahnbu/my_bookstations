# 광주시 조회 순차화 수정 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use `superpowers:subagent-driven-development` (recommended) or `superpowers:executing-plans` to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Cloudflare Worker 운영 조회에서 광주시 종이책 KOLAS와 시립소장 전자책을 같은 요청 안에서 병렬 호출하지 않고, `paper -> owned` 순서와 총 15초·개별 10초 예산으로 안정화한다.

**Architecture:** 광주시 종이책(`gwangjuPaper`)을 먼저 실행하고, 남은 총 예산 안에서 시립소장 전자책(`sirip owned`)을 실행한다. 시립구독 전자책, 경기도 전자도서관, 경기도교육청 전자책은 광주시 위험 endpoint가 아니므로 기존 병렬성을 최대한 유지한다. 저장 구조는 바꾸지 않고, 시립소장을 시도하지 못한 경우 기존 `siripEbook.errors.owned` 문자열에 “총 예산 부족으로 시립소장 조회 생략” 사유를 남긴다.

**Tech Stack:** Cloudflare Workers, TypeScript, Node.js `node:test`, Wrangler, Supabase

## 배경 근거

- 최종 진단 문서: [[20260707_04_cloudflare-first-광주시-수집-테스트결과]]
- Cloudflare 독립 Worker 측정에서 `pair-parallel`과 `pair-parallel-jitter`는 paper/owned 모두 5회 전체 timeout이었다.
- `paper-single`은 4/5 성공했고, 성공 시 약 1.9~2.4초였다.
- `pair-sequential`은 paper 4/5, owned 3/5 성공했다.
- 따라서 목표는 timeout 연장이 아니라 광주시 endpoint의 동시 연결을 제거하고, 전체 응답시간 상한을 유지하는 것이다.

## 정책

| 항목 | 정책 | 완료 기준 |
|---|---|---|
| 실행 순서 | `gwangjuPaper` 먼저, `sirip owned` 나중 | source test가 owned가 paper 완료 전 시작되지 않음을 확인 |
| 개별 timeout | paper 최대 10초, owned 최대 10초 | helper unit test가 `min(10초, 남은 총예산)`을 검증 |
| 총 예산 | paper + owned 합산 최대 15초 | unit test와 운영 측정에서 전체 POST가 약 16초 이내 |
| owned 생략 | 남은 예산이 1초 미만이면 owned 생략 | `siripEbook.errors.owned`에 생략 사유가 기록됨 |

```typescript
import { SiripEbookOwnedResult } from './types';

export const GWANGJU_GROUP_BUDGET_MS = 15000;
export const GWANGJU_ENDPOINT_TIMEOUT_MS = 10000;
export const MIN_GWANGJU_OWNED_ATTEMPT_MS = 1000;
export const SIRIP_OWNED_SKIPPED_BY_BUDGET_MESSAGE = '총 예산 부족으로 시립소장 전자책 조회 생략';

export function getRemainingBudgetMs(
  startedAtMs: number,
  totalBudgetMs = GWANGJU_GROUP_BUDGET_MS,
  nowMs = Date.now(),
): number {
  return Math.max(0, totalBudgetMs - (nowMs - startedAtMs));
}

export function getEndpointTimeoutMs(
  startedAtMs: number,
  totalBudgetMs = GWANGJU_GROUP_BUDGET_MS,
  endpointCapMs = GWANGJU_ENDPOINT_TIMEOUT_MS,
  nowMs = Date.now(),
): number {
  return Math.max(0, Math.min(endpointCapMs, getRemainingBudgetMs(startedAtMs, totalBudgetMs, nowMs)));
}

export function shouldAttemptOwnedLookup(remainingBudgetMs: number): boolean {
  return remainingBudgetMs >= MIN_GWANGJU_OWNED_ATTEMPT_MS;
}

export function createSiripOwnedSkippedResult(): SiripEbookOwnedResult {
  return {
    libraryName: '시립도서관 소장형 전자책',
    totalCount: 0,
    availableCount: 0,
    unavailableCount: 0,
    bookList: [],
    error: SIRIP_OWNED_SKIPPED_BY_BUDGET_MESSAGE,
  };
}
```

Add this helper immediately before `searchSiripEbookIntegrated`:

```typescript
function buildSiripEbookResult(
    searchTitle: string,
    ownedResults: PromiseSettledResult<SiripEbookOwnedResult>,
    subscriptionResults: PromiseSettledResult<SiripEbookSubsResult>
): SiripEbookResult {
    const ownedData = ownedResults.status === 'fulfilled' ? ownedResults.value : null;
    const subsData = subscriptionResults.status === 'fulfilled' ? subscriptionResults.value : null;

    const ownedError = ownedResults.status === 'rejected' ? (ownedResults.reason as Error).message : ownedData?.error;
    const subsError = subscriptionResults.status === 'rejected' ? (subscriptionResults.reason as Error).message : subsData?.error;

    const combinedBookList: SiripEbookBook[] = [
        ...(ownedData?.bookList || []),
        ...(subsData?.bookList || [])
    ];

    const totalCountOwned = ownedData?.totalCount || 0;
    const totalCountSubs = subsData?.totalCount || 0;
    const totalCountSummary = totalCountOwned + totalCountSubs;

    const availableCountOwned = ownedData?.availableCount || 0;
    const availableCountSubs = subsData?.availableCount || 0;
    const availableCountSummary = availableCountOwned + availableCountSubs;

    const flatResult: SiripEbookResult = {
        libraryName: '시립도서관 전자책',
        totalCountSummary,
        availableCountSummary,
        unavailableCountSummary: totalCountSummary - availableCountSummary,
        totalCountOwned,
        totalCountSubs,
        availableCountOwned,
        availableCountSubs,
        searchQuery: searchTitle,
        bookList: combinedBookList,
    };

    if (ownedError || subsError) {
        flatResult.errors = {};
        if (ownedError) flatResult.errors.owned = ownedError;
        if (subsError) flatResult.errors.subscription = subsError;
    }

    return flatResult;
}
```
