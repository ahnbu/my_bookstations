# BACKLOG — 후속 작업 후보

현재 작업 범위 밖이지만 나중에 가치 있는 작업을 기록합니다. 새 항목은 표 최하단에 추가합니다.

## BACKLOG 관리 규칙

### 추가 기준 (다음 조건 중 하나 이상)

- 지금 당장 실행하지 않지만 나중에 가치 있는 버그·개선·기술부채
- 발견된 문제 중 현재 작업 범위 밖인 것
- 다음 세션에서 이어받아야 할 미완료 작업

### 제외 기준 (추가하지 않음)

- 사소한 스타일 정리, 타입 힌트 추가 등 임팩트 낮은 것
- 이미 완료된 작업
- 즉시 실행해야 할 것 (바로 처리하거나 TODO 주석으로)

### 상태 표기 규칙

- **완료 항목**: 표에서 행을 삭제한다. 취소선(~~텍스트~~) 사용 금지. 히스토리는 git log로 추적.
- **보류 항목**: 항목명 앞에 `[보류]` 접두어를 붙인다. 명시적 지시가 있기 전까지 착수하지 않는다.
- **일반 항목**: 접두어 없음. 우선순위에 따라 착수 가능.

### ID 규칙

- 형식: `BL-MMDD-NN` (MM=월, DD=일, NN=당일 순번 01부터)
- 한 번 부여된 ID는 변경·재사용하지 않는다 (삭제해도 번호 유지)
- 외부 문서에서 참조 시: `BL-0412-01` 형태로 사용
