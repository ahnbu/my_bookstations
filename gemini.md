# Gemini 작업 규칙

이 문서는 Gemini 에이전트와의 협업을 위한 규칙을 정의합니다.

## 1. Git 커밋 규칙

- **커밋 메시지 제목은 반드시 한국어로 작성합니다.**
- 메시지 본문은 변경 사항을 명확히 알 수 있도록 상세히 기술합니다.

## 2. 작업 및 문서화 절차

- **선(先) 문서화, 후(後) 커밋 원칙을 준수합니다.**
- 코드 변경, 기능 추가, 리팩토링 등 모든 개발 작업 완료 후, 관련된 `docs` 폴더 내 문서를 최신 상태로 업데이트합니다.
- 모든 코드와 문서 변경사항이 완전히 반영되었는지 확인한 후, 마지막에 `git commit`을 실행합니다. 이를 통해 커밋 누락을 방지합니다.
